2. Critical System Configuration

The following configurations are mandatory to bypass LXC/Proxmox security restrictions and allow Docker to function correctly.

A. Docker Daemon Config (Target Machine)

File: /etc/docker/daemon.json
Purpose: Allows HTTP registry download AND fixes the "failed to mount" overlay error.

{
  "insecure-registries": ["192.168.1.7:5000"],
  "storage-driver": "fuse-overlayfs"
}

Note: Requires apt install fuse-overlayfs and a Docker restart.

B. Proxmox Host Configuration (For Target Container)

File: /etc/pve/lxc/1XX.conf (on Proxmox Host)
Purpose: Allows Docker to load security profiles and mount FUSE devices.

# Required for Docker Security Profiles
lxc.apparmor.profile: unconfined

# Required for fuse-overlayfs
lxc.mount.entry: /dev/fuse dev/fuse none bind,create=file 0 0

Alternatively, enable "FUSE" and "Nesting" in the Proxmox GUI Options.

3. Deployment Pipeline

The pipeline runs on `workflow_dispatch` and handles the full Build-to-Deploy lifecycle.

1

Checkout & Build

Runner checks out code and builds images using docker compose build.

2

Push to Local Registry

Images are tagged with 192.168.1.7:5000/... and pushed. Credentials are not required for this internal IP.

3

Transfer Configuration (SCP)

The docker-compose.prod.yml and .env are copied to the target (192.168.1.54).
Flag used: -o UserKnownHostsFile=/dev/null to bypass SSH host verification.

4

Remote Execution

SSH triggers docker compose pull followed by docker compose up -d.
Because we use `fuse-overlayfs`, the containers start successfully without overlay errors.

4. Troubleshooting Common Errors

Error Message Cause Solution
http: server gave HTTP response to HTTPS client Docker defaults to HTTPS, but local registry is HTTP. Add "insecure-registries": ["192.168.1.7:5000"] to daemon.json.
failed to mount ... fstype: overlay LXC containers cannot use the default Overlay2 driver. Install fuse-overlayfs and set "storage-driver": "fuse-overlayfs" in daemon.json.
fuse: device not found The container does not have access to /dev/fuse. Enable FUSE in Proxmox Container Options (Features).
apparmor_parser: Unable to replace "docker-default" Nested AppArmor conflict (Security Inception). Set lxc.apparmor.profile: unconfined on Proxmox Host, or apt purge apparmor inside the container.
Host key verification failed SSH unknown host fingerprint. Add flag -o UserKnownHostsFile=/dev/null to SSH/SCP commands.
